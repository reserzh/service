Docker Quick Reference

Important: Node modules are NOT installed on the host. All dev commands (pnpm dev, pnpm typecheck, etc.) must run inside the Docker container. 
# Start the container (auto-runs pnpm dev via entrypoint)
docker compose -f /Users/john/WEB/SERVICE/docker-compose.yml up -d

# Check logs
docker logs service-app --tail 20

# Exec into container
docker exec -it service-app zsh

# Restart (fixes EADDRINUSE port conflicts)
docker restart service-app

# Stop
docker compose -f /Users/john/WEB/SERVICE/docker-compose.yml down

# Run Claude Code inside container (sandboxed yolo mode)
claude --dangerously-skip-permissions

Useful Commands

Development (inside container)

pnpm dev                  # Start all apps (Turbo)
pnpm dev --filter back    # Start only back
pnpm dev --filter front   # Start only front
pnpm typecheck            # TypeScript check all packages
pnpm build                # Build all
pnpm lint                 # Lint all

Supabase (on host machine)

cd /Users/john/WEB/SERVICE
supabase start            # Start local Supabase
supabase stop             # Stop local Supabase
supabase status           # Show ports and URLs
supabase db reset         # Reset database to clean state

Mobile App (inside container at /workspace/apps/mobile)

npx expo start            # Start Expo dev server
npx expo start --clear    # Clear cache and start

Architecture Overview

SERVICE/
├── apps/
│   ├── back/           # Admin Dashboard — Next.js 16, port 3200
│   │   ├── src/app/(auth)/         # Login, register
│   │   ├── src/app/(dashboard)/    # 35+ admin pages
│   │   ├── src/app/api/v1/         # 50+ REST endpoints (mobile API)
│   │   ├── src/lib/services/       # Business logic layer
│   │   ├── src/lib/auth/           # Auth + RBAC (5 roles)
│   │   └── src/actions/            # Server actions
│   ├── front/          # Public Sites + Portal — Next.js 16, port 3201
│   │   ├── src/app/portal/         # Customer portal (auth, dashboard, jobs, invoices, ...)
│   │   ├── src/app/api/portal/     # Portal API routes
│   │   └── src/app/[slug]/         # Tenant websites + booking
│   └── mobile/         # Technician App — React Native / Expo
│       └── src/                    # Offline-first, field mode, voice notes
├── packages/
│   ├── api-types/      # Shared enums, models, constants (zero dependencies)
│   └── shared/         # Drizzle ORM schema, templates, website types
└── supabase/           # Local Supabase config (config.toml)

Key Architecture Decisions

AspectApproach
Multi-tenancyShared DB with tenant_id on all tables + RLS
AuthSupabase Auth — requireAuth() / requireApiAuth()
PermissionsRBAC with 5 roles: admin, office_manager, dispatcher, csr, technician
ServicesBusiness logic in src/lib/services/, shared by server actions + REST API
ORMDrizzle ORM, schema namespace fieldservice
ValidationZod v4
PaymentsStripe Connect (Checkout for portal payments)
EmailResend + template engine, Inbucket for local testing
Job NumbersTenant-scoped atomic sequences (tenant_sequences table)
